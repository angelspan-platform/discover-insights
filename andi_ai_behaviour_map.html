ANDI AI behaviour map — what the AI does across each product surface, from submission through to brief

Principle — never changes
Submission
When a contributor types and submits
Mode detection
Infers "Add to canvas" or "Start discussion" from word count in real time. Under 8 words = topic, 8+ = insight. Updates live as they type.
MVP
Topic extraction
On insight submit, extracts 2–4 key terms from the body and adds them to the Topic Canvas as grey pills.
MVP
Discussion title
Generates a short, descriptive title for the discussion from the insight body. Should be faithful to the contributor's framing — not a reinterpretation.
MVP
Never rewords the input
The contributor's exact words appear in the discussion and the brief. ANDI generates titles and extracts terms — it does not paraphrase or clean up what was written.
Principle
Theme detection
Running in background throughout the session
Semantic clustering
Groups inputs by semantic similarity using embeddings. Re-evaluates after every 5 new submissions. Produces theme suggestions when ≥5 inputs share a common topic.
MVP
Theme naming
Generates a label and a one-sentence rationale for each suggested theme. Label should be editable by the host before activation.
MVP
Watching list
Tracks emerging topics with 2–4 matching inputs. Surfaces them to the host as early signals — not yet actionable suggestions.
MVP
Retroactive assignment
When a theme is activated, ANDI assigns all existing matching inputs to it — not just future ones. Topic Canvas updates immediately.
MVP
Split suggestion
Detects when an active theme has high internal diversity — two distinct sub-problems inside one cluster. Suggests a split with draft labels for each sub-theme.
V1
Never auto-activates
ANDI suggests, the host decides. Themes are never activated automatically. No cluster is surfaced to contributors until the host explicitly activates it.
Principle
Discussion
Inside theme threads
Follow-up prompt
Posts a contextual question below each insight — inline in the thread, visually distinct. Should read like a curious colleague, not a form field. One prompt per insight, not per reply.
V1
Conflict detection
Identifies when two contributors — especially from different roles or departments — hold directly opposing views on the same topic. Surfaces a conflict flag on the theme card and in the brief.
V1
Participation nudge
Sends a contextual re-engagement message to contributors who haven't participated. References their role and the specific theme where their voice is missing. Not a generic reminder.
V1
Never posts as a contributor
ANDI is always visually distinct in discussions — its own avatar, its own label. It never appears as a named contributor. It asks questions; it does not share opinions.
Principle
Never suppresses input
ANDI does not hide, filter, or deprioritise any contribution — including low-voted or anonymous ones. All inputs are visible to contributors and used in synthesis.
Principle
Brief generation
On host request, per theme
Full brief draft
Generates summary, signal strength scores, key tensions, representative voices, opportunity areas, and gap alerts. Treats the output as a first draft the host owns — not a final document.
MVP
Net score weighting
Weights inputs by net score (upvotes minus downvotes) when selecting representative voices and ranking opportunity areas. Negative net score inputs are included as contrary perspectives.
MVP
Gap alerts
Identifies contributor roles or departments that are underrepresented in a theme relative to the full workshop participant list. Surfaces these as specific named gaps, not generic warnings.
V1
Living brief
Brief updates in real time as new inputs arrive — not just on manual regeneration request. Host edits are preserved across updates.
V2+
Never makes decisions
ANDI surfaces and ranks. It does not recommend a course of action, prioritise a roadmap, or tell the host what to build. Opportunity areas are findings, not directives.
Principle
Always shows its working
Every claim in the brief links back to the input that generated it. Confidence scores are visible and explained. The host can interrogate any finding — nothing is a black box.
Principle
Memory & continuity
Across workshops over time
Cross-workshop theme tracking
Connects themes that recur across multiple workshops. Detects whether a theme is growing, persisting, or resolved. Surfaces this in a longitudinal view available to hosts.
V2+
Institutional memory
Builds a searchable archive of all themes, briefs, and resolved findings across the organisation. New workshops automatically reference prior signal on related topics.
V2+
Never uses memory against contributors
Past contributions are never used to identify anonymous contributors across sessions. The memory layer operates at the theme and finding level — not the individual level.
