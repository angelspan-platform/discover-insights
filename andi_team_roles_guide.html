ANDI Experiment — Team Roles Guide

What each role is doing, thinking about, and producing at every stage

Core principle: Humans source the data and review AI output. The AI does the reading, parsing, and structuring. No team member should be manually reading through transcripts or raw data.

Preparation Set up the test environment. This step is the same for both Round 1 and Round 2. No reading of source data required.
Before any stage Setup
Do
  • Collect raw data files — SPL transcripts and/or Reddit exports, unedited
  • Write a one-sentence workshop prompt framing the problem for each community
  • Create a Claude Project per dataset. Upload files to Project Knowledge. Add the workshop prompt as system context
  • Prepare a blank evaluation log with one row per MVP behavior
Consider
  • Is the workshop prompt specific enough to give the AI meaningful context, or so broad it could justify any theme?
  • Are all relevant files uploaded? Are any transcripts or posts missing?
Produce
  • One Claude Project per dataset, loaded and ready
  • Workshop prompt written for each community
  • Blank evaluation log
Do
  • Review the MVP behaviors in the framework so you know what to watch for
  • Confirm Claude Projects are correctly set up before running anything
Consider
  • What does a good theme suggestion look like vs. a vague or premature one?
Produce
  • Confirmation that Projects are ready and prompts are understood
Do
  • For SPL: write down the themes you already know came out of that engagement — before the experiment starts
  • Define what "good enough to show a client" means to you before seeing any AI output
Consider
  • Setting expectations before the experiment prevents rationalising weak output after the fact
Produce
  • Written expectation of SPL themes, recorded before any prompt is run
Do
  • Set up the evaluation log
  • Read the pass/fail criteria for each MVP behavior in the framework
Consider
  • What specific evidence would constitute a pass vs. a fail for each behavior?
Produce
  • Evaluation log populated and ready to record
Stage 2 Theme Identification — The AI reads the raw data and surfaces themes. Humans review and approve the output, not the source material.
Theme Identification MVP
Active in Preparation only
Do
  • Run the clustering prompt against the full dataset. Do not pre-process or summarize the data.
  • Capture the full AI output verbatim. Do not filter or interpret it.
  • Hand the raw output directly to the Moderator.
Consider
  • Did the AI return labeled themes with supporting rationale, or just a list?
  • Did anything unexpected appear in the output?
Produce
  • Raw AI theme output — unedited, handed to Moderator
Do
  • Run the clustering prompt in passes — ask the AI to update suggestions as it processes more data
  • Record every AI response verbatim by pass
  • Run at least one pass with sparse or conflicting data to test edge case behavior
Consider
  • Are theme suggestions stable across passes, or changing significantly each time?
  • Are labels specific ("Invoice-to-Cash Handoff") or generic ("Process Issues")?
  • Is the AI surfacing themes prematurely before it has enough data?
Produce
  • Verbatim log of AI theme suggestions, organized by pass
  • Note of when suggestions first appeared and when they stabilized
Do
  • Review AI-suggested themes — not the source data.
  • For each theme: Approve, Rename, or Reject.
  • Write one sentence of rationale per decision.
Consider
  • For SPL: do the AI's themes match what you wrote down before the experiment?
  • Are any themes missing that you expected to see?
Produce
  • Approved theme list with one-sentence rationale per decision
Do
  • Review AI-suggested themes — not the source data
  • For each suggestion: Approve, Rename, or Reject
  • Flag any gaps — themes you expected to see that the AI did not surface
Consider
  • For SPL: do the AI's themes match your pre-experiment expectations?
  • Would contributors recognize their input in this theme?
  • Has the AI subtly reframed what was said, or is the label accurate?
  • Are any two themes the same problem described differently?
Produce
  • Approved theme list with written rationale for every decision
Do
  • In the framework, mark Semantic Clustering, Theme Naming, and Watching List as: Observed / Issues Noted / Not Run — for this dataset.
  • Note any failure modes you observed.
Produce
  • Framework rows updated for this dataset
Do
  • Record when the first theme suggestion appeared and when suggestions stabilized
  • Log all moderator decisions and their rationale
  • Note any failure modes: premature themes, generic labels, expected themes not surfaced
Consider
  • Is the AI's behavior consistent with the framework pass criteria?
  • What would need to change in the prompt to address failures?
Produce
  • Completed Stage 2 rows in the evaluation log — Pass / Partial / Fail with observations
Stage 3 Discussion Deepening — AI generates follow-up questions to deepen signal within approved themes. Humans assess question quality. (V1 scope)
Discussion Deepening V1
Not active at this stage
Do
  • For a sample of approved theme inputs, run the follow-up prompt.
  • Log each question generated alongside the input that produced it.
Produce
  • Paired log: source input → AI-generated question
Do
  • For each approved theme, run the follow-up prompt against a full set of inputs
  • Record every generated question verbatim alongside its source input
  • Run the participation nudge prompt for at least one theme with a missing role
Consider
  • Is the AI generating one question per input, or more than one?
  • Does the question reference something specific from that input?
Produce
  • Verbatim log of all AI questions paired with their source inputs
Do
  • For each generated question, rate it: Specific / Generic / Misleading.
  • Note any patterns across the sample.
Produce
  • Rated question log — one rating per question
Do
  • Review each generated question — not the source input
  • Rate each question: Specific and useful / Generic / Misleading
Consider
  • If a real participant received this question, would they give a more useful answer than they already did?
  • Could this question have been written without reading the specific input at all?
Produce
  • Rated question log with one assessment per question
Do
  • Mark Follow-up Prompt and Participation Nudge in the framework for this dataset.
Produce
  • Framework rows updated for Stage 3 behaviors
Do
  • Review question ratings and identify patterns
  • Flag any questions that could apply to any input regardless of content
Consider
  • What is the ratio of specific to generic questions?
  • Does quality vary depending on how detailed the original input was?
Produce
  • Completed Stage 3 rows in the evaluation log
Stage 4 Brief Synthesis — AI produces a structured brief from approved themes. Humans assess whether the output is accurate, specific, and usable.
Brief Synthesis MVP
Not active at this stage
Do
  • Run the brief generation prompt on all approved themes.
  • Capture the full output verbatim. Hand directly to Moderator.
Produce
  • Full brief output, unedited
Do
  • Run the brief generation prompt on all approved themes
  • Run a second pass with intentionally thin data to test whether the AI flags its own uncertainty or overstates confidence
  • Record all outputs verbatim
Consider
  • Does the brief cover all required sections?
  • Is the structure consistent across themes?
Produce
  • Full brief per approved theme
  • Thin-data brief for SPL comparison
Do
  • Read the brief. Does it match the themes you approved in Stage 2?
  • Flag any quoted voice you cannot trace to a specific input — automatic failure.
  • Assess: would you present this to a client?
Produce
  • Assessment per brief: Approve / Rework / Would not use — one sentence each
Do
  • Read each brief as if presenting it to a client
  • Check: do the themes match what was approved in Stage 2?
  • Flag any quoted voice that cannot be traced to a specific input — automatic failure
Consider
  • Would I put my name on this and present it in a client meeting?
  • Are the opportunity areas actionable, or restatements of the theme?
  • For SPL: does this match what you know the engagement actually produced?
Produce
  • Assessment per brief: Approve for client use / Requires rework / Would not use — with one sentence of reasoning
Do
  • Mark Brief Generation and Gap Alerts in the framework for this dataset.
  • Write a one-paragraph summary: what held, what didn't, for this dataset.
  • Use the summary to inform whether Round 2 targeted tests are needed on any behavior.
Produce
  • Framework rows updated for Stage 4
  • Dataset summary — used to decide Round 2 scope
Do
  • Verify brief themes match Stage 2 approved themes
  • Score each section against the framework pass criteria
  • For SPL: compare brief against the expected themes written before the experiment
Consider
  • Is the brief technically accurate but too generic to be useful?
  • Does the thin-data version overstate confidence or appropriately flag its limits?
Produce
  • Completed Stage 4 rows in the evaluation log
  • Overall pass/fail summary across all MVP behaviors — used to refine prompts before the next dataset or Round 2
Throughout Documentation — Ongoing across all stages, both rounds. The evaluation log is the primary output of the experiment. AI responses are raw material; the log is the product.
All stages Ongoing
Active in Preparation only
Do
  • Record all AI responses verbatim — never summarize or paraphrase
  • Note which prompt and which dataset produced each response
  • Log unexpected results immediately — do not wait until end of stage
Consider
  • An unexpected result is as valuable as a pass — record it with the same care
Produce
  • Verbatim log of all AI responses, organized by stage and dataset
Do
  • Write rationale for every decision at the time it is made — not retrospectively
  • Do not revise decisions after seeing later outputs
Consider
  • Decisions made in the moment are the data — they must reflect genuine judgment, not hindsight
Produce
  • Contemporaneous decision log with rationale per decision
Do
  • Update the framework status after each stage — do not batch updates at the end
  • Write one sentence per behavior: what happened, not just pass or fail
  • After Round 1 on all datasets: identify which behaviors need Round 2 targeted tests
Consider
  • Vague log entries produce vague prompt improvements — be specific
  • Are patterns consistent across datasets, or does the AI behave differently on SPL vs. Reddit?
Produce
  • Fully populated evaluation log after each stage
  • Round 1 complete: shortlist of behaviors flagged for Round 2 targeted testing
